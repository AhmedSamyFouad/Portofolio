TC_API_AUTH_001
OAuth 2.0 Token Refresh & Expiry
API Security Critical Automated ✓ Pass

Verify that the OAuth 2.0 token refresh mechanism correctly issues a new access token using a valid refresh token, and that expired tokens are properly rejected with 401 Unauthorized.

  • 1
    Initial LoginPOST /auth/login with valid credentials. Capture access_token (15min TTL) and refresh_token (7day TTL).
  • 2
    Access Protected ResourceGET /api/user/profile with valid access_token. Verify 200 OK and correct user data.
  • 3
    Wait for Token ExpiryWait 15 minutes or manipulate system clock to exceed access_token TTL. Attempt resource access.
  • 4
    Refresh Token RequestPOST /auth/refresh with valid refresh_token. Verify new access_token and refresh_token are returned.
  • 5
    Expired Refresh TokenWait 7 days or revoke refresh token. Attempt refresh. Verify 401 with error: "invalid_grant".
  • 6
    Reuse Refresh TokenAttempt to reuse the same refresh_token after successful refresh. Verify 401 and token revocation.
Expected Result
Access token expires correctly after 15min. Refresh token generates new pair. Reused refresh tokens are revoked. Expired refresh returns 401 with clear error message.
TC_DB_ETL_002
Database ETL Referential Integrity
Data Integrity High Priority Automated ✓ Pass

Verify that the ETL migration from legacy Oracle DB to PostgreSQL maintains referential integrity, data type accuracy, and zero record loss for 5M+ customer transaction records.

  • 1
    Source Count ValidationExecute COUNT(*) on Oracle source tables: customers, orders, order_items. Record baseline counts.
  • 2
    Trigger ETL PipelineExecute migration script. Monitor logs for errors, warnings, or truncation events.
  • 3
    Target Count ValidationExecute COUNT(*) on PostgreSQL target tables. Verify exact match with source counts.
  • 4
    Referential Integrity CheckRun FK constraint validation: every order_items.order_id must exist in orders.id. Verify 0 orphans.
  • 5
    Data Type AccuracyVerify DECIMAL(10,2) amounts match exactly. Check TIMESTAMP precision to millisecond.
  • 6
    Checksum ValidationGenerate MD5 checksums on sensitive columns (email, phone) in both DBs. Verify 100% match.
Expected Result
All 5M+ records migrated with zero loss. Referential integrity maintained. Data types preserved. Checksums match 100%. ETL log shows zero errors.
TC_PERF_LOAD_003
Checkout Flow Under Peak Load
Performance Critical k6 Script ✓ Pass

Validate that the checkout API maintains p95 response time under 800ms and 0% error rate when handling 10,000 concurrent users during Black Friday simulation.

  • 1
    Environment SetupDeploy k6 cloud agents across 3 regions. Configure test data: 10k valid user sessions, product SKUs.
  • 2
    Ramp-up PhaseGradual ramp: 0 → 2,000 users over 5min. Monitor DB connection pool, CPU, and memory baseline.
  • 3
    Steady State LoadMaintain 10,000 concurrent users for 15min. Execute full checkout flow: cart → shipping → payment → confirm.
  • 4
    Spike TestSudden spike to 15,000 users for 2min. Measure recovery time and error rate. Verify circuit breaker activation.
  • 5
    Ramp-down & AnalysisGradual decrease to 0 users. Generate Grafana dashboard. Verify p95 < 800ms, error rate 0%, no memory leaks.
Expected Result
p95 response time < 800ms throughout test. 0% error rate. No connection pool exhaustion. Memory usage stable. Circuit breaker functions correctly under spike.
TC_SEC_XSS_004
Stored XSS in Product Review
Security Critical Manual + Tool ✓ Fixed

Verify that the product review submission properly sanitizes user input to prevent stored XSS attacks, and that the WAF blocks malicious payloads before persistence.

  • 1
    Baseline ReviewSubmit legitimate review: "Great product!". Verify it displays correctly in product page and admin panel.
  • 2
    Basic XSS PayloadSubmit: <script>alert('xss')</script>. Verify WAF blocks request with 403 or input is sanitized.
  • 3
    Encoded PayloadSubmit URL-encoded variant: %3Cscript%3Ealert('xss')%3C%2Fscript%3E. Verify same protection applies.
  • 4
    HTML Event HandlerSubmit: <img src=x onerror=alert('xss')>. Verify image tag is stripped or event handler removed.
  • 5
    Persistence CheckIf payload bypasses, verify it does NOT execute in admin panel, email notifications, or mobile app.
  • 6
    DOM Context ValidationInspect browser DevTools. Confirm no script injection in DOM. Verify CSP headers are present.
Expected Result
All XSS payloads are blocked or sanitized. No script execution occurs in any rendering context. WAF logs show blocked attempts. CSP headers enforced.
